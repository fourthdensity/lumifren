│ 13 ## Reporting a Vulnerability                                                                │
│ 14                                                                                             │
│ 15 We take the security of our users and their data seriously. If you believe you have found a │
│    security vulnerability in Lumifren, please report it to us as soon as possible.             │
│ 16                                                                                             │
│ 17 **Please do not report security vulnerabilities via public GitHub issues.**                 │
│ 18                                                                                             │
│ 19 ### How to report                                                                           │
│ 20                                                                                             │
│ 21 1. **Email:** Please send an email to                                                       │
│    [security@fourthdensity.ai](mailto:security@willtyler.tech)                                                            │
│ 22 2. **Private Reporting:** Alternatively, you can use GitHub's [Private Vulnerability        │
│    Reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporti │
│    ng-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerabili │
│    ty) feature if enabled on this repository.                                                  │
│ 23                                                                                             │
│ 24 ### What to expect                                                                          │
│ 25                                                                                             │
│ 26 - **Acknowledgment:** You will receive an acknowledgment of your report within 48-72 hours. │
│ 27 - **Investigation:** We will investigate the issue and may contact you for further details. │
│ 28 - **Resolution:** If the vulnerability is confirmed, we will work on a fix and release a    │
│    new version.                                                                                │
│ 29 - **Public Disclosure:** Once a fix is released and users have had time to update, we will  │
│    coordinate public disclosure if appropriate.                                                │
│ 30                                                                                             │
│ 31 ### Guidelines                                                                              │
│ 32                                                                                             │
│ 33 - Provide a clear description of the vulnerability.                                         │
│ 34 - Include steps to reproduce the issue (proof-of-concept).                                  │
│ 35 - Mention the impacted versions and components.                                             │
│ 36 - Please give us reasonable time to investigate and fix the issue before sharing details    │
│    publicly.                                                                                   │
