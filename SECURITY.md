# Security Policy

## Supported Versions

Lumifren is currently in active development. We prioritize security updates for the latest stable releases.

| Version | Supported          |
| ------- | ------------------ |
| 4.2.x   | :white_check_mark: |
| 4.1.x   | :white_check_mark: |
| < 4.1   | :x:                |

## Reporting a Vulnerability

We take the security of our users and their data seriously. If you believe you have found a security vulnerability in Lumifren, please report it to us as soon as possible.

**Please do not report security vulnerabilities via public GitHub issues.**

### How to report

1. **Email:** Please send an email to [security@fourthdensity.ai](mailto:security@fourthdensity.ai)
2. **Private Reporting:** Alternatively, you can use GitHub's [Private Vulnerability Reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) feature if enabled on this repository.

### What to expect

- **Acknowledgment:** You will receive an acknowledgment of your report within 48-72 hours.
- **Investigation:** We will investigate the issue and may contact you for further details.
- **Resolution:** If the vulnerability is confirmed, we will work on a fix and release a new version.
- **Public Disclosure:** Once a fix is released and users have had time to update, we will coordinate public disclosure if appropriate.

### Guidelines

- Provide a clear description of the vulnerability.
- Include steps to reproduce the issue (proof-of-concept).
- Mention the impacted versions and components.
- Please give us reasonable time to investigate and fix the issue before sharing details publicly.

Thank you for helping keep Lumifren secure!
